## 🏗️ Architecture Rules

### Module Structure
Every module MUST follow this pattern:
```lua
-- ModuleName.lua
-- Short description
-- vX.X.X

local Dependency = require(script.Parent.Dependency)
local ModuleName = {}

-- Public API functions only
function ModuleName.Init() end
function ModuleName.Start() end

return ModuleName
```

### Mandatory Rules
1. **No globals** — all state is module-local or passed via arguments
2. **WorldConfig is the single config file** — never hardcode values in modules
3. **Seed-deterministic** — every procedural call must accept a `seed` param and produce identical output for the same seed
4. **No `wait()` or `delay()`** — use `RunService.Heartbeat` or `task.spawn`/`task.delay`
5. **No `print()` in production** — use `warn()` for errors only; wrap debug prints in `if WorldConfig.Debug then`
6. **No `Instance.new()` inside tight loops** — batch-build parts, parent last
7. **Module names = PascalCase**, function names = camelCase or PascalCase, locals = camelCase
8. **Every module that touches Players must handle PlayerRemoving** to clean up state

### Folder Layout
```
src/
  init.server.lua         ← entry point, boots WorldGenerator
  WorldGenerator.lua      ← master orchestrator
  WorldConfig.lua         ← ALL config values live here
  BiomeResolver.lua       ← biome sampling
  ChunkHandler.lua        ← chunk build/destroy lifecycle
  OreGenerator.lua        ← ore vein placement
  RiverCarver.lua         ← river erosion
  DungeonGenerator.lua    ← dungeon room + corridor layout
  LootTable.lua           ← weighted loot generation
  MobSpawner.lua          ← biome-aware NPC spawning
  WeatherManager.lua      ← server weather state machine
  WeatherClient.lua       ← client visual effects
  SeedPersistence.lua     ← DataStore seed save/load
  StreamingManager.lua    ← chunk streaming radius
  AssetPlacer.lua         ← tree/rock/foliage decoration
  StructurePlacer.lua     ← village/structure placement
  QuestSystem.lua         ← procedural quest generation
  AdminPanel.lua          ← in-game debug/admin GUI (server)
  LODManager.lua          ← level-of-detail chunk simplification
docs/
  architecture.md         ← system diagram + data flow
  api-reference.md        ← public API for each module
rojo/
  default.project.json
```

## 🔧 Development Workflow

### When adding a new feature:
1. Add config values to `WorldConfig.lua` first
2. Create the module in `src/`
3. Require it in `WorldGenerator.lua` and call `.Start(seed)` or `.Init()` in `WorldGenerator.Init()`
4. Add the module entry to `rojo/default.project.json`
5. Update `CHANGELOG.md` with the version bump
6. Update `docs/api-reference.md` with public functions

### Commit format:
```
feat(vX.X): short description
fix(module): what was broken
refactor(module): what changed
docs: what was documented
```
