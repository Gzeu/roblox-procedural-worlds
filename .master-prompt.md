## 🏗️ Architecture Rules

### Module Structure
Every module MUST follow this pattern:
```lua
-- ModuleName.lua
-- Short description
-- vX.X.X

local Dependency = require(script.Parent.Dependency)
local ModuleName = {}

function ModuleName.Init() end
function ModuleName.Start() end

return ModuleName
```

### Mandatory Rules
1. **No globals** — all state is module-local
2. **WorldConfig is the single config file** — never hardcode values
3. **Seed-deterministic** — every procedural call accepts `seed` and is reproducible
4. **No `wait()` or `delay()`** — use `RunService.Heartbeat` or `task.spawn`/`task.delay`
5. **No `print()` in production** — use `warn()` only inside `if WorldConfig.Debug then`
6. **No `Instance.new()` inside tight loops** — batch-build, parent last
7. **Module names = PascalCase**, locals = camelCase
8. **Every module touching Players must handle PlayerRemoving**

## 🔧 Development Workflow

1. Add config to `WorldConfig.lua` first
2. Create module in `src/`
3. Require + call in `WorldGenerator.Init()`
4. Add entry to `rojo/default.project.json`
5. Update `CHANGELOG.md`
6. Update `docs/api-reference.md`

### Commit format
```
feat(vX.X): description
fix(module): what was broken
refactor(module): what changed
docs: what was documented
```
